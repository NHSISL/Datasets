## Summary

This release introduces several improvements to data consistency, traceability, and address formatting within the EMIS OLIDS transformation models. Notably, a new timestamp column has been added to enhance record tracking, and address string formatting has been improved for better UPRN matching. SQL refactoring ensures more reliable data types and clearer field naming, supporting downstream compatibility and maintainability. The release also includes a version update to OLIDS_Share V1.1.0.

### Dependencies

- Repository dependency on **OLIDS_Share** upgraded from `v1.0.1` to [v1.1.0](../1_OLIDS_Share/v1.1.0.md)

### New Features

- Add LDS_TRANSFORM_DATETIME to PC_EMIS_Location_Contact and reorder columns for consistency. [PR [#101](https://github.com/NHSISL/LondonDataservices.Transform.EMIS_OLIDS/pull/101)]  
  > ✨ ***Feature:*** Adds a transformation timestamp to each record, improving traceability and auditability of data changes.

### Improvements

- Remove repeated commas from address string to improve UPRN matching. [PR [#100](https://github.com/NHSISL/LondonDataservices.Transform.EMIS_OLIDS/pull/100)]  
  > 🎯 ***Impact:*** Produces cleaner, more standardized address outputs, enhancing the accuracy of UPRN matching and downstream address processing.

### Refactoring

- Refactor SQL for PC_EMIS_Referral_Request and PC_EMIS_Observation to enforce type consistency and clarity. [PR [#95](https://github.com/NHSISL/LondonDataservices.Transform.EMIS_OLIDS/pull/95)]  
  > ♻️ ***Refactor:*** Ensures boolean and datetime fields use correct data types and improves naming clarity, reducing risk of data errors and aligning with best practices.
- Update revision for OLIDS_Share to V1.1.0. [PR [#102](https://github.com/NHSISL/LondonDataservices.Transform.EMIS_OLIDS/pull/102)]  
  > ♻️ ***Refactor:*** Updates the project version to reflect new changes, supporting version tracking and deployment workflows.
