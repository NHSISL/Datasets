## Summary

This release delivers key updates to data sharing and privacy controls. The sharing table configuration has been updated to support new recipients, expanding the scope of data distribution. Additionally, the masking policy for hashed data has been improved to prevent double hashing, ensuring data consistency and simplifying downstream processing. These changes enhance both the flexibility and reliability of the platform.

### New Features
- Update sharing table configuration to include PSEUDO and PID data sharing with 5 ICBs and LAP. [PR #10](https://github.com/NHSISL/LondonDataServices.OLIDS_Share/pull/10)  
  > ✨ ***Feature:*** Enables broader data sharing capabilities, supporting new organizational recipients for improved collaboration.

### Bug Fixes
- Amend hashed masking policy to avoid hashing values that are already hashed, preventing double hashing issues. [PR #11](https://github.com/NHSISL/LondonDataServices.OLIDS_Share/pull/11)  
  > 🐞 ***Fix:*** Ensures that hashed data, such as postcodes, are not hashed multiple times, maintaining data integrity and alignment with lookup tables. 
  > <br>Fixes Raised UAT BUG: [#AB29809/SEL: Postcode to Postcode_HASH join returning no results](https://dev.azure.com/NELAnalytics/LondonDataService/_workitems/edit/29809)
