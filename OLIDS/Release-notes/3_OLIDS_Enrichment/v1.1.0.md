## Summary

This release introduces several enhancements and new features to the OLIDS Enrichment project, focusing on improved developer experience, expanded data model coverage, and pipeline reliability. Key highlights include robust developer environment setup scripts, new DBT models for location-level contact data sharing, and ingestion objects for UPRN and NDOO data. Additional improvements streamline pipeline configuration and update data models for greater consistency and maintainability.

### Dependencies

- Repository dependency on **OLIDS_Share** upgraded from `v1.0.1` to [v1.1.0](../1_OLIDS_Share/v1.1.0.md)
- Repository dependency on **EMIS_OLIDS** upgraded from `v1.0.0` to [v1.1.0](../2_EMIS_OLIDS/v1.1.0.md)


### New Features

- Add models for location-level contact data ([Location_Contact_PID](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/42), [Location_Contact_Pseudo](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/42)) for sharing with consumers, exempt from masking and row access policies. *[PR [#42](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/42)]*  
  > ✨ ***Feature:*** Enables sharing of location-level contact data in both PID and pseudonymized formats, supporting broader data access needs.

- Adding UPRN and NDOO ingestion objects, plus tweaks to PDS. *[PR [#43](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/43)]*  
  > ✨ ***Feature:*** Expands the platform's ability to ingest and process UPRN and NDOO datasets, enhancing data enrichment capabilities.
  > Contributes towards resolving known issue/defect [#AB KI-01: static PDS, UPRN and NDOO records:](https://dev.azure.com/NELAnalytics/LondonDataService/_workitems/edit/29779)

### Improvements

- Add developer environment helpers for venv management, including robust detection, setup automation, and improved test script reliability. *[PR [#24](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/24)]*  
  > 🎯 ***Impact:*** Streamlines developer onboarding, reduces environment setup errors, and ensures consistent test execution across platforms.

- UPRN pipeline updates: reordered parameters for clarity, added GitHub authentication environment variables, and improved SQL consistency. *[PR [#38](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/38)]*  
  > 🎯 ***Impact:*** Improves pipeline reliability, security, and maintainability by clarifying configuration and ensuring correct authentication and SQL logic.

- Remove redundant `ORGANISATION_ID` and rename `RECORDED_DATE` to `RECORDED_DATETIME` in models for schema alignment. *[PR [#33](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/33)]*  
  > 🎯 ***Impact:*** Ensures data model consistency and prepares for future package updates.

### Refactoring

- Add root-level virtual environment to `.gitignore` to prevent accidental versioning of local environments. *[PR [#41](https://github.com/NHSISL/LondonDataservices.Transform.OLIDS_Enrichment/pull/41)]*  
  > ♻️ ***Refactor:*** Keeps the repository clean by ignoring local virtual environment directories, reducing clutter and potential merge conflicts.
